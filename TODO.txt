# Flight Search GUI - To-Do List

## Current Implementation Standpoint

Current state:
- Project idea is defined.
- No implementation exists yet.
- Required core features are known:
  - Search cheap flights by origin, destination, time/date, and maximum price.
  - Show results in a simple GUI.
  - Save selected routes for price tracking.
  - Store price history over time.
  - Plot price timeline for a selected route.

Next step:
- Build the project structure and implement a minimal working prototype with mock flight data before connecting a real API.

## Definition of Done for First Version

The first version is complete when:

[ ] The app starts with python main.py.
[ ] The user can enter origin, destination, date, and maximum price.
[ ] The app displays matching flight offers.
[ ] The user can save a route for tracking.
[ ] The app stores price history locally.
[ ] The user can manually update the tracked price.
[ ] The app plots the price history of a selected route.
[ ] API keys are not hardcoded.
[ ] The app works with mock data even without internet/API access.
