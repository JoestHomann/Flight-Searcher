# Flight Search GUI - Instructions and Requirements

## 1. Project Goal

Build a Python application with a simple graphical user interface that can search for cheap flights on the internet, filter them by destination, date/time window, and maximum price, track selected flight prices over time, and plot the historical price development for a selected route.

The application should be useful for quickly checking whether flights from a chosen origin to a chosen destination are currently below a desired price threshold.

---

## 2. Main Features

### 2.1 Flight Search

The user shall be able to search for flights using:

- Origin airport or city
- Destination airport or city
- Departure date
- Optional return date
- One-way or round-trip selection
- Flexible date range, if supported by the flight API
- Maximum acceptable price
- Number of passengers
- Currency selection, for example EUR
- Optional filters:
  - Direct flights only
  - Maximum number of stops
  - Preferred airlines
  - Departure time range
  - Arrival time range

The application shall display matching flight offers in a simple table.

Each result should show:

- Airline
- Origin airport
- Destination airport
- Departure time
- Arrival time
- Duration
- Number of stops
- Price
- Currency
- Booking/source link if available

---

## 3. Price Tracking

The application shall allow the user to save a search route for repeated price checks.

A tracked route should include:

- Origin
- Destination
- Departure date or date range
- Return date or date range, if applicable
- Maximum target price
- Currency
- Search frequency setting, for example manual, daily, or hourly
- Last checked timestamp
- Lowest known price
- Current known price

The application shall store historical prices locally.

Price history should include:

- Timestamp of check
- Origin
- Destination
- Departure date
- Return date, if applicable
- Airline
- Price
- Currency
- Number of stops
- Source/API provider

---

## 4. Price Timeline Plot

The user shall be able to select a tracked route and view a graph showing price development over time.

The graph should show:

- X-axis: date/time of price check
- Y-axis: flight price
- One line for the cheapest found price per check
- Optional markers for lowest price
- Optional horizontal line for the user's target price

The plot can be implemented with matplotlib and embedded into the GUI.

---

## 5. Graphical User Interface

The GUI should be simple and functional.

Recommended GUI framework:

- Tkinter for a simple first version
- Optional later upgrade: PySide6 or PyQt6

Main GUI sections:

### Search Tab

Contains:

- Origin input field
- Destination input field
- Departure date input
- Return date input
- One-way / round-trip selector
- Maximum price input
- Currency selector
- Search button
- Results table

### Tracking Tab

Contains:

- List of saved tracked routes
- Add current search to tracking button
- Remove tracked route button
- Manual update/check button
- Display last checked price
- Display lowest recorded price

### Price Graph Tab

Contains:

- Dropdown/list for selecting tracked route
- Plot button
- Embedded price timeline plot

### Settings Tab

Contains:

- API provider settings
- API key input or environment variable information
- Default currency
- Default origin airport
- Data storage path
- Search interval settings

---

## 6. Flight Data Source

The application should not scrape websites directly unless the website explicitly allows it.

Preferred approach:

- Use an official flight search API
- Example providers:
  - Amadeus for Developers
  - Kiwi/Tequila API
  - Skyscanner API if access is available
  - Aviationstack or similar APIs, depending on available features

API keys should not be hardcoded into the source code.

API keys should be loaded from:

- Environment variables
- A local .env file
- A config file that is excluded from Git

Example environment variables:

FLIGHT_API_PROVIDER=amadeus
AMADEUS_CLIENT_ID=your_client_id
AMADEUS_CLIENT_SECRET=your_client_secret
DEFAULT_CURRENCY=EUR

## 10. Functional Requirements

REQ-001:
The application shall allow the user to search flights by origin, destination, and date.

REQ-002:
The application shall allow the user to define a maximum acceptable price.

REQ-003:
The application shall display flight results in a GUI table.

REQ-004:
The application shall store selected routes for future price tracking.

REQ-005:
The application shall save price history locally.

REQ-006:
The application shall plot the price history for a selected tracked route.

REQ-007:
The application shall support at least one real flight API provider.

REQ-008:
The application shall include a mock API provider for development and testing without using real API calls.

REQ-009:
The application shall handle API errors gracefully.

REQ-010:
The application shall not expose API keys in the source code.

---

## 11. Non-Functional Requirements

NFR-001:
The application should be simple to install and run locally.

NFR-002:
The GUI should be understandable without reading the source code.

NFR-003:
The search should not freeze the GUI while API requests are running.

NFR-004:
The application should work on Windows.

NFR-005:
The database should be stored locally.

NFR-006:
The application should be modular so that different flight API providers can be added later.

NFR-007:
The application should avoid direct website scraping unless explicitly allowed by the provider.

NFR-008:
The application should be robust against missing internet connection, invalid API keys, and empty API responses.

---

## 12. Error Handling Requirements

The application should show user-friendly messages for:

- Missing origin or destination
- Invalid date format
- Invalid price input
- Missing API key
- API rate limit reached
- No flights found
- Network error
- Database error

The application should log technical error details internally.

---

## 13. MVP Definition

The first working version should include:

- Simple Tkinter GUI
- Manual flight search
- Mock provider for testing
- One real API provider
- SQLite storage
- Ability to save a route for tracking
- Manual price check for tracked routes
- Price history plot with matplotlib

Automatic background tracking and notifications can be added later.

---
